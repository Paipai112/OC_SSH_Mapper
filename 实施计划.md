# OpenClaw SSH 转发工具 - 重构与增强实施计划

## 1. 项目概述

### 1.1 项目背景
OpenClaw SSH 转发工具是一个用于管理服务器 OpenClaw WebUI 访问的桌面应用程序。用户通过 SSH 隧道将服务器上的 OpenClaw 服务映射到本地，从而通过浏览器访问。

### 1.2 当前状态分析

**技术栈：**
- GUI 框架：Tkinter + ttk
- SSH 库：Paramiko
- 系统托盘：pystray
- 图像处理：PIL (Pillow)
- 配置存储：JSON 文件

**现有功能：**
- SSH 本地端口转发（-L 等效功能）
- 基础连接配置（IP、端口、用户名、密码）
- 密码记忆功能（Base64 编码）
- 系统托盘支持（隐藏到托盘）
- 自动打开浏览器
- 端口占用检测与释放

**代码质量问题：**
| 问题 | 严重性 | 说明 |
|------|--------|------|
| 单文件 480 行 | 中 | 违反单一职责原则 |
| Base64 加密密码 | 高 | 无安全性，可轻易解码 |
| 无输入验证 | 高 | 可能导致注入攻击 |
| 无自动重连机制 | 中 | 断线后需手动重连 |
| UI 简陋 | 低 | 使用原生 Tkinter 组件 |
| 无日志系统 | 中 | 问题排查困难 |
| 无配置预设管理 | 低 | 多服务器场景需手动修改 |

## 5. 技术选型

### 5.1 依赖库

| 库名 | 版本 | 用途 | 替代方案 |
|------|------|------|----------|
| moderntkinter | 5.2+ | 现代化 UI 框架 | customtkinter |
| paramiko | 3.4+ | SSH 连接 | asyncssh |
| pystray | 0.19+ | 系统托盘 | pywin32 (仅 Windows) |
| cryptography | 42+ | AES 加密 | pycryptodome |
| Pillow | 10+ | 图像处理 | - |
| pywin32 | 306+ | Windows API（开机启动） | - |

### 5.2 requirements.txt

```
moderntkinter>=5.2.0
paramiko>=3.4.0
pystray>=0.19.0
cryptography>=42.0.0
Pillow>=10.0.0
pywin32>=306; sys_platform == 'win32'
```

### 5.3 技术决策说明

**为什么选择 ModernTkinter？**
- 基于 Tkinter，学习曲线低
- 原生支持深色模式
- 组件美观现代
- 向后兼容现有 Tkinter 代码

**为什么选择 cryptography 而非 pycryptodome？**
- 更活跃的维护
- 更安全的默认配置
- 更好的 API 设计
- Fernet 模式适合密码加密
